This project analyzes a year's worth of pizza sales data to uncover **business insights, customer behavior, and product performance**.

Using **SQL for data analysis**, **Excel for preprocessing**, and **Power BI for visualization**, I built an interactive dashboard that helps stakeholders make **data-driven decisions**.

**Business Problem**
 The business needed clarity on:
* Which pizzas drive the most revenue?
* When are peak sales periods?
* What products underperform?
* How customer ordering behavior impacts revenue?

 **Tools & Technologies**
*  SQL – Data querying & KPI calculations
*  Excel – Data cleaning & preprocessing
*  Microsoft Power BI – Dashboard creation & visualization

  **Project Files**
* `pizza_sales_excel_file.xlsx` → Raw dataset
* `pizzadb.sql` → SQL queries for KPIs and analysis
* `Pizza_Sales_Dashboard.pbix` → Power BI dashboard file
* `Pizza_Sales_Dashboard` → Exported dashboard report
* `README.md` → Project documentation

  **Key KPIs**
* **Total Revenue:** 818K
* **Total Orders:** 213K+
* **Total Pizzas Sold:** 495K+
* **Average Order Value:** 38.31
* **Average Pizzas per Order:** 2.32

  **Key Insights** 
* **Revenue Driver:** Classic pizzas contribute the highest share of total sales
*  **Peak Time:** Orders spike on **Fridays & weekends**
*  **Seasonality:** Sales peak in **July & January**, dip in **September–October**
*  **Customer Behavior:** Average order = **2.32 pizzas (~$38)**
*  **Size Preference:** **Large pizzas (~46%) dominate sales**
*  **Optimization Opportunity:** Several pizzas consistently rank in bottom 5

  **Dashboard Highlights**
* KPI Cards (Revenue, Orders, AOV, Quantity)
* Daily & Monthly Sales Trends
* Best vs Worst Selling Pizzas
* Sales by Category & Size
* Interactive Filters (Date, Category)

  **Business Recommendations**
*  Target **weekend promotions** (especially Friday evenings)
*  Boost sales during **low-performing months** with offers
*  Promote **large-size pizzas** via combos or upselling
*  Reassess or improve **low-performing menu items**
